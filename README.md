# Інформаційна система «Клуб любителів туризму»

## Опис проєкту
Інформаційна система призначена для автоматизації діяльності туристичного клубу. Система забезпечує підтримку бази даних країн, тематичних турів та туристичних агенцій, дозволяючи користувачам підбирати оптимальні варіанти відпочинку згідно з їхніми вподобаннями та запланованими датами.

---

## Ролі користувачів та доступ

В системі розрізняють чотири основні групи користувачів, кожна з яких має свій рівень доступу та функціонал:

1. **Адміністрація клубу**
   * Модерація списку країн та тематик турів.
   * Контроль діяльності туристичних агенцій.
   * Формування аналітичних звітів (популярні напрямки, статистика відвідувань).
   * Робота з архівом завершених турів.

2. **Туристичні агенції**
   * Додавання та оновлення інформації про власні туристичні тури.
   * Вказівка кількості вільних місць (ставок) у турах.
   * Управління датами заїздів та актуалізація цін.
   * Робота із заявками від туристів.

3. **Турист (Користувач сервісу)**
   * Пошук та фільтрація турів за країною, тематикою та датою.
   * Перегляд інформації про надійність агенцій.
   * Вибір туру та бронювання місць згідно з власними вподобаннями.

4. **Технічний персонал**
   * Підтримка цілісності бази даних (PostgreSQL).
   * Налаштування серверної частини (C++ Backend).
   * Моніторинг роботи системи в Docker-контейнерах та керування доступами.

---

## Функціональні можливості (Звіти та Списки)

Система надає можливість формування специфічних списків та посадових розкладів турів:

* **Списки турів:** перелік турів з вільними місцями (з урахуванням дат, до яких бронь є актуальною).
* **Гарячі пропозиції:** тури, дата вильоту яких запланована на найближчий час.
* **Архів подорожей:** інформація про завершені тури для аналізу вподобань.
* **Рейтинг агенцій:** список партнерів, що працюють на ринку понад 10 років (ветерани ринку).
* **Бюджетні варіанти:** фільтрація турів за вартістю, що нижча за середню.
* **Календар подорожей:** формування індивідуального графіку для туриста без накладок за датами.

## Технологічний стек
* **Backend:** C++ (Framework: Crow / Drogon)
* **Database:** PostgreSQL
* **Infrastructure:** Docker & Docker Compose
* **Frontend:** HTML5, CSS3, JavaScript (через браузер)
